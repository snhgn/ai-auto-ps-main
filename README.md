# ai-auto-ps

一个基于轻量化开源视觉大模型的 AI 自动 P 图/调色工具，支持常见图片和视频格式，并提供图形化操作界面。

## 已实现功能

- 选用开源轻量模型：`HuggingFaceTB/SmolVLM2-500M-Video-Instruct`
  - 开源仓库：<https://github.com/huggingface/smollm>
  - **新增**：支持升级到更强的模型（LLaVA-1.6-13B，通过环境变量 `AI_AUTO_PS_USE_LARGE_MODEL=1` 启用）
  - 自动fallback：模型加载失败时自动降级到SmolVLM2
- 自动内容分析与风格路由
  - 模型可用时走模型推理
  - 模型不可用时自动回退到本地启发式分析，保证流程可用
  - **新增**：多维度图像特征分析（场景识别、主体检测、光照估算）
- 支持高级人像修图能力
  - 皮肤美化：磨皮 / 美白 / 祛痘 / 红润
  - 五官塑形：瘦脸 / 大眼 / 窄鼻 / 下巴
  - 细节润色：亮眼 / 唇色
  - 支持风格预设自动参数 + 手动参数覆盖
- 预设多种修图调色风格
  - `portrait_soft` - 人像柔和
  - `landscape_vivid` - 风景鲜艳
  - `night_clarity` - 夜景清晰
  - `cinematic` - 电影感
  - `food_fresh` - 美食新鲜
  - `clean_natural` - 自然清透
- **🆕 多方案生成**
  - 一张图片同时生成 3-4 个不同修改方向供对比选择
  - 8 种方案模板：
    - `color_correction` - 色温纠正（恢复自然色彩）
    - `skin_tone_enhance` - 肤色优化（增加气色，仅人像）
    - `contrast_pop` - 对比强化（增加视觉冲击力）
    - `vibrance_boost` - 活力增强（提升饱和度）
    - `portrait_retouch` - 人像精修（全面精修，仅人像）
    - `cinematic_grade` - 电影分级（电影风格）
    - `detail_sharpen` - 细节锐化（增强清晰度）
  - 智能场景推荐：根据图像类型（人像/风景/美食/夜景）推荐最适合的方案组合
- 支持常见图片格式（基础）
  - `jpg/jpeg/jpe/jfif/png/bmp/dib/tif/tiff/webp/heic/gif/ppm/pgm/pbm/pnm`
- 支持常见视频格式（基础）
  - `mp4/mov/mkv/avi/webm/m4v/flv/wmv/mpeg/mpg/ts/m2ts/mts/3gp/3g2/ogv/ogm/asf/vob`
- 支持环境可选图片格式（按 Pillow 编解码能力自动启用）
  - `avif/jp2/j2k/jpf/jpx/jxl/ico/icns/pcx/dds`
- 图形化界面（Gradio）
  - 支持一次上传多张图片（视频单文件处理）
  - 自动/手动风格选择
  - **新增**：界面分区重构（方案预览 / 点评记忆 / 重新生成），减少功能堆叠感
  - **新增**：处理模式选择（单方案/多方案）
  - **新增**：方案数量调整（1-4 个）
  - 亚洲人像精修调整板块（可折叠）
  - 人像参数预设包切换（自然韩系 / 清透日系 / 轻欧式）
  - 处理前后图片预览（批量对比）
  - 输出处理结果、风格决策依据、双轮完整性检查结果

## 🎯 新增功能：多版本对比与迭代反馈系统

### ✨ 核心特性

- **AI 推荐理由**：每个修图方案都包含详细的推荐理由，解释为什么 AI 选择这个方案
- **多版本对比**：同时展示多个修图方案，支持详细对比
- **用户反馈机制**：
  - 三级评价系统（满意👍 / 一般🤷 / 不满意👎）
  - 详细反馈文本输入
  - 支持申请使用更强 AI 模型
- **迭代优化建议**：AI 根据反馈自动生成优化建议
  - 模型升级建议
  - 参数调整建议
  - 风格改变建议
- **会话管理**：追踪每个修图流程的完整信息
- **双模型协作分析**：主模型 + 辅助模型共同决策风格
- **偏好记忆库**：根据用户点评沉淀偏好，冲突项按最新时间戳覆盖
- **重生成预览闭环**：点评后重生成可直接预览并继续点评，且每轮都基于原图避免滤镜叠加
- **完整报告导出**：支持导出 Markdown 格式的修图报告

### 📊 使用流程

```
1. 上传图片 → AI 分析
      ↓
2. 展示多个方案 + 推荐理由
      ↓
3. 用户选择方案并反馈
      ├─ 给出评价
      ├─ 输入反馈内容
      └─ 可选：请求强模型
      ↓
4. AI 分析反馈生成迭代建议
      ↓
5. 用户可继续：
      ├─ 基于反馈重新生成
      ├─ 导出完整报告
      └─ 多轮迭代优化
```

## 当前任务流程

1. 上传媒体文件（可批量上传图片）
2. 媒体类型识别（扩展名 -> MIME -> 解码探测）
3. 风格路由（手动风格直通；自动风格走模型/启发式）
4. 应用调色并输出文件 + 前后预览
5. 双轮完整性检查并返回结果

## 流程优化说明

- 模型顾问实例复用：避免每次请求重复初始化模型，提升连续处理流畅性。
- 媒体识别链路统一：先快速判断扩展名，再回退 MIME 与解码探测，降低误判。
- 视频处理资源安全释放：写入器与读取器统一在 `finally` 中释放，减少中途异常时资源泄漏风险。
- 格式扩展机制升级：基础格式稳定支持，可选格式根据运行环境自动增量启用。

## 运行方式

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python ai_auto_ps.py
```

打开浏览器访问：<http://127.0.0.1:7860>

## Windows 一键启动

项目根目录已提供统一启动脚本：`start.bat`

### 正常启动（推荐）

在资源管理器中双击 `start.bat` 或在终端运行：
```bash
start.bat
```

- 首次运行会自动创建 `.venv` 并安装依赖
- 服务就绪后自动打开浏览器（默认地址：<http://127.0.0.1:7860>）
- 后续启动会跳过依赖检查，快速启动

### 调试启动

需要查看详细日志时，运行：
```bash
start.bat debug
```

显示详细的调试信息，包括主机、端口、URL等参数

### 环境变量配置（可选）

可先设置环境变量自定义运行参数：

- `AI_AUTO_PS_HOST`（默认 `127.0.0.1`）
- `AI_AUTO_PS_PORT`（默认 `7860`）
- `AI_AUTO_PS_OPEN_BROWSER`（默认 `1`，设为 `0` 时不自动打开浏览器）
- `AI_AUTO_PS_USE_LARGE_MODEL`（默认 `0`，设为 `1` 时使用更强的LLaVA模型）
- `AI_AUTO_PS_FORCE_PIP`（默认 `0`，设为 `1` 时强制重装依赖）
- `AI_AUTO_PS_BOOTSTRAP_PIP`（默认 `0`，设为 `1` 时升级 `pip/setuptools/wheel`，适用于需要重建安装工具链的场景）

## 双轮完整性检查

应用内置 `double_check_implementation()`，每次处理后都会进行两轮检查：

1. 第一轮（配置检查）
   - 是否已配置轻量开源模型
   - 是否已配置多风格预设
   - 是否同时具备图片与视频格式支持
2. 第二轮（运行时检查）
   - 运行一个最小样例图像的分析与调色流程
   - 校验返回风格与预设一致
   - 再次校验格式声明规范

两轮都通过才返回“检查通过”。
